# Control de accesos · prototipo navegable

Prototipo público con datos ficticios para comunicar una propuesta a Gerencia TI, Ciberseguridad y Cumplimiento.

## Problema

Los antecedentes de usuarios y accesos suelen vivir en planillas y fuentes separadas. Investigar una señal exige cruces manuales y consume tiempo.

## Propuesta

Un recorrido navegable de tres pasos:

1. Detectar una señal en el resumen.
2. Investigar los accesos de una persona.
3. Priorizar aplicaciones según su nivel de riesgo.

Las tarjetas, navegación, búsquedas, filtros y estados son interactivos. Todos los nombres y registros de esta versión son ficticios.
